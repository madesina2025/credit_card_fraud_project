# Database Schema

## Credit Card Fraud Detection Database

---

## Overview

The database stores fraud prediction results generated by the API and consumed by the real-time dashboard.

Current database:

- SQLite

Future production database:

- PostgreSQL

---

# Main Table

## fraud_predictions

Stores all transaction prediction records.

---

# Table Structure

| Column | Data Type | Description |
|----------|---------|-------------|
| id | Integer | Unique transaction record ID |
| customer_id | String | Customer identifier |
| merchant | String | Merchant name |
| country | String | Transaction country |
| channel | String | Transaction channel |
| transaction_amount | Float | Transaction amount |
| prediction | Integer | 1 = Fraud, 0 = Genuine |
| risk_label | String | Fraud or Genuine |
| fraud_probability | Float | Model fraud probability |
| risk_category | String | Critical, High, Medium, Low |
| recommended_action | String | Operational action |
| created_at | DateTime | Prediction timestamp |

---

# Risk Categories

| Risk Category | Condition |
|---------------|----------|
| Critical | Fraud Probability ≥ 0.90 |
| High | Fraud Probability ≥ 0.70 |
| Medium | Fraud Probability ≥ 0.40 |
| Low | Fraud Probability < 0.40 |

# Future PostgreSQL Schema

```sql
CREATE TABLE fraud_predictions (
    id SERIAL PRIMARY KEY,
    customer_id VARCHAR(50),
    merchant VARCHAR(100),
    country VARCHAR(20),
    channel VARCHAR(50),
    transaction_amount NUMERIC(12,2),
    prediction INTEGER,
    risk_label VARCHAR(20),
    fraud_probability NUMERIC(6,4),
    risk_category VARCHAR(20),
    recommended_action TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
